# QA Directory

Test cases, test data, and QA validation artifacts.

## Structure

```
qa/
├── test-cases/     # Manual and automated test case definitions
├── test-data/      # Test data sets
└── test-reports/   # QA test execution reports
```

## Workflow

1. Create test cases from requirements/tickets
2. Execute tests (manual or automated)
3. Document results in test reports
4. Link test failures to bug tickets
